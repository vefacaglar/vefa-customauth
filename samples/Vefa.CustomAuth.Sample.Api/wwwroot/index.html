How Vefa.CustomAuth SSO Flow Operates

1
Authentication Server
The Client WebApp redirects users to the SSO Server (port 5175) to log in via Authorization Code Flow + PKCE. Upon success, an Access Token (JWT) is issued.
2
Token Handshake
The WebApp acquires the JWT and attaches it to the HTTP request as an Authorization: Bearer <token> header when calling this API.
3
Cryptographic Verification
This Resource Server (port 5098) extracts the JWT and cryptographically validates its signature against the public JWKS keys fetched from the SSO Server.
